# Toast Alert Monitor v0.14.1

Мониторинг уведомлений Telegram через центр уведомлений Windows, фильтрация сообщений, голосовое оповещение (TTS), ретрансляция в MQTT (Meshtastic) и **дедупликация** сообщений.

## Возможности

- Прослушивание Toast-уведомлений Telegram Desktop
- Фильтрация по отправителям и ключевым словам (три списка `A/B/C`, логика И/ИЛИ)
- Дедупликация сообщений (хэш + расстояние Левенштейна) с настраиваемым окном и порогом
- Озвучивание сообщений через синтезатор речи Windows (TTS) с поддержкой аббревиатур
- **Выбор TTS голоса из установленных в системе (горячая клавиша F2) с запоминанием выбора**
- **Тестовое меню (горячая клавиша F3) для проверки голоса, MQTT и имитации уведомлений**
- Отправка отфильтрованных сообщений в MQTT-брокер (JSON для Meshtastic)
- Автоматическое переподключение к MQTT при обрыве связи
- Удаление обработанных уведомлений (immediate/on_exit/never)
- Горячие клавиши для управления (включая **F1** для справки)
- Защита конфигурационного файла от повреждения (атомарная запись, бэкап)
- Восстановление доступа к уведомлениям после сна/пробуждения системы
- Опциональный запрет перехода компьютера в сон
- Мгновенное закрытие программы по крестику окна (без задержек)

## Требования

- Windows 10/11 (версия 19041 или выше)
- Установленный Telegram Desktop (работает через уведомления)
- .NET 8.0 Runtime (для самодостаточной сборки – не требуется)

## Установка и запуск

1. Скачайте и распакуйте архив с программой (или скачайте один EXE‑файл).
2. Запустите `ToastAlert.exe`.
3. При первом запуске программа запросит доступ к уведомлениям Windows – разрешите.
4. При необходимости отредактируйте `config.json` (создастся автоматически).
5. Горячая клавиша `R` позволяет перезагрузить конфиг без остановки.

## Конфигурация (config.json)

Все настройки хранятся в файле `config.json`. При изменении настроек **не перезаписывайте файл вручную во время работы программы** – используйте горячую клавишу `R` для перезагрузки.

### Секция `Фильтрация`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `AllowedSenders` | массив строк | [] | Разрешённые отправители. |
| `Keywords` | массив строк (СписокА) | [] | Основные ключевые слова. |
| `СписокБ` | массив строк | [] | Второй список ключевых слов. |
| `СписокС` | массив строк | [] | **Приоритетные слова** – при наличии сообщение всегда проходит фильтр. |
| `Оператор` | string | "И" | Логика между списками A и B: `"И"` или `"ИЛИ"`. |
| `BlacklistedSenders` | массив строк | [] | Чёрный список отправителей. |
| `BlacklistedKeywords` | массив строк | [] | (не используется) |
| `CaseSensitive` | bool | false | Чувствительность к регистру. |
| `PartialMatchSenders` | bool | true | Частичное совпадение отправителя (contains). |
| `PartialMatchKeywords` | bool | true | Частичное совпадение ключевых слов. |

### Секция `Дедупликация`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `Enabled` | bool | true | Включить дедупликацию. |
| `TimeWindowSeconds` | int | 120 | Время хранения сообщений в буфере (сек). |
| `SimilarityThresholdHigh` | double | 0.95 | Порог, при котором сообщение считается дублем (95%). |
| `SimilarityThresholdLow` | double | 0.85 | Порог для предупреждения (не блокирует). |
| `BufferMaxSize` | int | 50 | Максимум сообщений в буфере. |
| `NormalizeTime` | bool | true | Заменять временные метки на `<TIME>` при сравнении. |
| `UseLevenshtein` | bool | true | Использовать расстояние Левенштейна (иначе только хэш). |
| `LogDuplicates` | bool | true | Выводить информацию о дублях в консоль. |

### Секция `ГолосовоеОповещение` (добавлен параметр `VoiceName`)

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| ... | ... | ... | ... (все остальные параметры без изменений) |
| `VoiceName` | string | `null` | Имя выбранного голоса (например, `"Microsoft Irina Desktop"`). Если задано, при запуске выбирается этот голос, иначе русский голос или системный по умолчанию. |

## Горячие клавиши (по умолчанию)

| Клавиша | Действие |
|---------|----------|
| `S` | Показать статистику |
| `C` | Показать текущую конфигурацию |
| `M` | Вкл/Выкл звук TTS |
| `+` / `-` | Увеличить/уменьшить громкость TTS |
| `R` | Перезагрузить `config.json` |
| `Q` | Вкл/Выкл MQTT |
| `D` | Очистить буфер дедупликации |
| `F1` / `H` | Показать справку |
| `F2` | Показать список доступных TTS голосов, переключить голос (выбор запоминается) |
| `F3` | Открыть тестовое меню (голос, MQTT, имитация уведомления) |
| `Esc` | Остановить программу (корректное завершение) |
| `Ctrl+C` | Альтернативная остановка |
| (крестик окна) | Мгновенное завершение (без задержек) |

## Логирование и файлы

- `telegram_alerts.log` – основной лог (с временными метками).
- `processed.json` – ID обработанных уведомлений.
- `config.json.bak` – резервная копия конфигурации.
