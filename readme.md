# Toast Alert Monitor v0.14.5

Мониторинг уведомлений **любых приложений Windows** через центр уведомлений, фильтрация сообщений, голосовое оповещение (TTS), ретрансляция в MQTT (Meshtastic) и **дедупликация** сообщений.

## Возможности

- Прослушивание Toast-уведомлений **любых приложений** (Telegram, Discord, браузеры и др.) – настраивается белый список `AllowedApps`
- Отдельная утилита **LearningMode** (F4) для автоматического сбора имён активных приложений
- Фильтрация по отправителям и ключевым словам (три списка `A/B/C`, логика И/ИЛИ)
- **Дедупликация** сообщений (хэш + расстояние Левенштейна) с настраиваемым окном и порогом
- Озвучивание сообщений через синтезатор речи Windows (TTS) с поддержкой аббревиатур
- **Выбор TTS голоса** из установленных в системе (F2), запоминание в `config.json`
- **Ручной ввод сообщения** (F12) – отправка произвольного текста в TTS и MQTT
- **Тестовое меню** (F3): проверка голоса, MQTT, имитация уведомления
- Отправка отфильтрованных сообщений в MQTT-брокер (JSON для Meshtastic)
- Автоматическое переподключение к MQTT при обрыве связи
- Удаление обработанных уведомлений (immediate/on_exit/never)
- Защита конфигурационного файла от повреждения (атомарная запись, бэкап, устойчивость к ошибкам)
- Восстановление доступа к уведомлениям после сна/пробуждения системы
- Опциональный запрет перехода компьютера в сон
- **Автоматический запуск в Windows Terminal** (если установлен) – цветные эмодзи, корректная работа мыши (настраивается)

## Требования

- Windows 10/11 (версия 19041 или выше)
- Установленный **любой** источник Toast-уведомлений (Telegram Desktop, Discord, браузеры и т.д.)
- .NET 8.0 Runtime (для самодостаточной сборки – не требуется)
- Для наилучшего отображения рекомендуется установить [Windows Terminal](https://aka.ms/terminal) (бесплатно)

## Установка и запуск

1. Скачайте и распакуйте архив с программой (или скачайте один EXE‑файл)
2. Запустите `ToastAlert.exe`
3. При первом запуске программа запросит доступ к уведомлениям Windows – разрешите
4. При необходимости отредактируйте `config.json` (создастся автоматически)
5. Горячая клавиша `R` позволяет перезагрузить конфиг без остановки

## Конфигурация (config.json)

### Секция `Мониторинг`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `CheckIntervalSeconds` | int | 2 | Интервал проверки (сек) |
| `StartMinimized` | bool | true | Запуск в свёрнутом окне |
| `AllowedApps` | массив строк | `["Telegram"]` | Белый список приложений, чьи уведомления обрабатывать. Если пусто – все приложения. |
| `DeleteMode` | string | `"immediate"` | Удаление уведомлений: `never`, `immediate`, `on_exit` |
| ... остальные параметры без изменений | ... | ... | ... |

### Секция `Фильтрация`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `AllowedSenders` | массив строк | [] | Разрешённые отправители (названия каналов/чатов) |
| `Keywords` | массив строк | [] | Основные ключевые слова (список A) |
| `СписокБ` | массив строк | [] | Географические/вторичные ключевые слова (список B) |
| `СписокС` | массив строк | [] | **Приоритетные слова** – при наличии сообщение всегда проходит фильтр |
| `Оператор` | string | `"И"` | Логика между списками A и B: `"И"` или `"ИЛИ"` |
| `CaseSensitive` | bool | false | Чувствительность к регистру |
| `PartialMatchSenders` | bool | true | Частичное совпадение имени отправителя |
| `PartialMatchKeywords` | bool | true | Частичное совпадение ключевых слов |

### Секция `Дедупликация`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `Enabled` | bool | true | Включить дедупликацию |
| `TimeWindowSeconds` | int | 120 | Время хранения сообщений в буфере (сек) |
| `SimilarityThresholdHigh` | double | 0.95 | Порог, при котором сообщение считается дублем (95%) |
| `SimilarityThresholdLow` | double | 0.85 | Порог для предупреждения (не блокирует) |
| `BufferMaxSize` | int | 50 | Максимум сообщений в буфере |
| `NormalizeTime` | bool | true | Заменять временные метки на `<TIME>` при сравнении |
| `UseLevenshtein` | bool | true | Использовать расстояние Левенштейна |
| `LogDuplicates` | bool | true | Выводить информацию о дублях в консоль |

### Секция `ГолосовоеОповещение`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `Enabled` | bool | true | Включить TTS |
| `VoiceName` | string | `null` | Имя выбранного голоса (например, `Microsoft Irina Desktop`). Сохраняется при выборе через F2 |
| `TtsVolume` | int | 30 | Громкость (0‑100) |
| `SpeakOnlyOnKeywords` | bool | true | Озвучивать только сообщения, прошедшие фильтр ключевых слов |
| `MinDelayBetweenMessagesMs` | int | 500 | Задержка между сообщениями (мс) |
| ... остальные параметры скорости, суффикса и т.д. | ... | ... | ... |

### Секция `Дополнительно`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `EnableHotkeys` | bool | true | Включить горячие клавиши |
| `MqttEnabled` | bool | true | Глобальное включение MQTT (наряду с `MQTT.Enabled`) |
| `RememberVolume` | bool | true | Запоминать изменённую громкость TTS в конфиге |
| `PreventSleep` | bool | false | Запретить переход компьютера в сон во время работы |
| `UseWindowsTerminal` | bool | true | Автоматически запускать программу в Windows Terminal (если установлен) |
| ... остальные параметры (`HotkeyStats`, `HotkeyConfig`, `DebugMode` и т.д.) работают как в предыдущих версиях | ... | ... | ... |

### Секции `MQTT`, `ПроверкаСвязи`, `Звуки`, `Логирование`, `ОчисткаТекста`, `Аббревиатуры`, `Приоритеты` – без изменений.

## Горячие клавиши (по умолчанию)

| Клавиша | Действие |
|---------|----------|
| `S` | Показать статистику |
| `C` | Показать **полную конфигурацию** (все параметры) |
| `M` | Вкл/Выкл звук TTS |
| `+` / `-` | Увеличить/уменьшить громкость TTS |
| `R` | Перезагрузить `config.json` |
| `Q` | Вкл/Выкл MQTT |
| `D` | Очистить буфер дедупликации |
| `H` / `F1` | Показать справку |
| `F2` | Список доступных голосов TTS / переключить голос |
| `F3` | Тестовое меню (голос, MQTT, имитация уведомления) |
| `F4` | Запустить **LearningMode** (сбор приложений) |
| `F12` | Ручной ввод сообщения (озвучить + отправить в MQTT) |
| `Esc` / `Ctrl+C` | Остановить программу |

## Дополнительные утилиты

### LearningMode (сбор приложений)

Запускается по `F4` или из командной строки (`ToastAlert.LearningMode.exe`).  
Собирает имена приложений, от которых поступают уведомления, и сохраняет их в `discovered_apps.txt`.  
Пользователь может скопировать нужные имена в секцию `AllowedApps` конфига.

## Рекомендации

- Для наилучшего отображения установите **Windows Terminal** из Microsoft Store – программа автоматически запустится в нём (если параметр `UseWindowsTerminal` не отключён). В нём эмодзи цветные, правый клик не мешает.
- Если Windows Terminal не установлен, программа работает в обычной консоли.

## Логирование и файлы

- `telegram_alerts.log` – основной лог (с ротацией по размеру)
- `processed.json` – ID обработанных уведомлений
- `config.json.bak` – резервная копия конфигурации
- `discovered_apps.txt` – создаётся LearningMode
